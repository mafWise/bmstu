Для запуска всего цикла, начиная от сбора данных до построения графиков, требуется запустить сркрипт go.sh

Для запуска промежуточного процесса сначала требуется создать соответствующую папку (build_apps.sh -> ./apps | testing.sh -> ./data | make_preproc.py -> preproc | make_postproc.py -> graphs) и потом запустить скрипт

Для всех скриптов, кроме go.sh, необходимы позиционные аргументы:
1. Минимальное количество элементов
2. Максимальное количество элементов
3. Шаг измерений 

При запуске скрипта update_data.sh требуется ввод позиционных аргументов: 
1. Номер файла с методом реализации алгоритма (1/2/3)
2. Уровень оптимизации (O1/O2/O3/O0/Os)
3. Количество элементов
4. Количество кругов тестирования

За валидность данных, введенных при запуске скрипта отдельного процесса, отвечает вызывающая сторона
